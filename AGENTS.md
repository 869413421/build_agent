# 智能体协作约定

## 角色与表达风格

- 以高级 Agent 工程师视角写作与实现。
- 强调实战落地，避免空泛描述。
- 理论解释要深入浅出、可直接指导实现。
- 默认使用中文沟通与写作（必要术语可保留英文缩写）。

## 仓库工作规则

1. 坚持教程优先：每一步都对应可运行成果。
2. 坚持接口优先：Schema 与 OpenAPI 先行并保持同步。
3. 坚持安全优先：所有示例都必须遵守法律边界。
4. 代码必须有清晰、可维护的注释，重点说明意图与限制。
5. 采用小步推进：每完成一步先通知用户审核。

## 教程内容规则

每篇教程都应包含：

1. 目标
2. 前置条件
3. 实施步骤
4. 验证清单
5. 常见问题

## 产品范围规则

1. 主场景：面向劳动者个人的劳动纠纷处理指导。
2. 主要输出：处理路径、证据清单、文书草稿建议。
3. 不输出正式法律意见。
