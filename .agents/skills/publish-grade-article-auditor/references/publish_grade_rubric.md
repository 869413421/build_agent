# Publish Grade Rubric

## 1. 结构流畅度

1. 关键认知必须出现在实现步骤之前。
2. 章节内必须存在清晰“先面后点”路径。
3. 长段落应拆分，段间要有承接句。

## 2. 主要代码讲解深度

对每段主要代码都必须覆盖：

1. 主流程拆解（按步骤）
2. 成功链路示例
3. 失败链路示例
4. 图示（流程图或时序图）
5. 工程取舍与边界

## 3. 工程可执行性

1. 文件路径真实可点击。
2. 命令可复制运行，PowerShell 可执行。
3. 有最小验证路径和常见错误排查。

## 4. 代码保真

1. 不删减任何原有代码块。
2. 允许新增代码块，不允许减少已有代码块。
3. 大文件节选必须给“进入源码复制完整代码”提醒。
