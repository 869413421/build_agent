# AI 任务执行规则

## 强制对齐

执行任何任务前，必须先阅读：

1. `NORTH_STAR.md`
2. `AI_TASK_GUARDRAILS.md`
3. `AGENTS.md`
4. `PROJECT_STATUS.md`

## 硬约束

1. 所有协作沟通与教程默认使用中文。
2. 全仓文件统一为 UTF-8（无 BOM），禁止混用本地编码。
3. 输出必须保持在法律信息辅助边界内，不提供正式法律意见。
4. 法律相关输出必须包含免责声明。
5. 每次只实现一个框架组件，不允许跨组件大步提交。
6. 每步必须同时提交：代码 + 测试 + 教程。
7. 代码必须包含清晰、明确的关键注释，解释意图与边界条件。
8. 未经用户审核“通过”，禁止进入下一步。

## 质量门禁

1. 功能变更必须同步更新测试。
2. API/Schema 与实现必须保持一致。
3. 设计变化必须记录到 `docs/architecture/`。
4. 会话结束前必须更新 `PROJECT_STATUS.md` 与 `HANDOFF_CHECKLIST.md`。

## 完成定义（DoD）

1. 本地可运行。
2. 对应测试通过。
3. 对应教程可复现。
4. 安全边界未被破坏。
