# AI 任务执行规则

## 强制对齐

在执行任何实现任务前，必须先对齐以下文件：

1. `NORTH_STAR.md`
2. 本文件
3. `AGENTS.md`

## 硬约束

1. 所有输出必须保持在“法律信息辅助”边界内。
2. 法律相关输出必须包含免责声明。
3. 优先可测试、可复现、可维护的实现，不做炫技式实现。
4. 检索驱动结论必须保留可追溯引用。
5. 禁止隐藏规则或未文档化行为。
6. 所有协作沟通默认使用中文。
7. 代码必须包含清晰明确的关键注释，重点解释设计意图与边界条件。
8. 每完成一个可独立验收的小步，必须先通知用户审核，再进入下一步。

## 质量门禁

1. 功能变更必须同步更新测试。
2. API 契约（Schema/OpenAPI）与实现必须保持一致。
3. 行为设计发生变化时，必须在 `docs/architecture/` 记录决策。

## 完成定义（DoD）

1. 本地可运行。
2. 测试通过。
3. 文档更新完成。
4. 安全边界未被破坏。
