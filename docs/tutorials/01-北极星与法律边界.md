## 实施步骤

1. 先写项目一句话定位：
   - 这是一个“法律信息辅助系统”，不是“法律意见系统”。
2. 固化 v1 目标：
   - 用户 60 分钟跑通 MVP。
   - 输出处理路径、证据清单、文书草稿建议。
3. 固化 v1 非目标：
   - 不保证胜诉概率。
   - 不替代律师。
4. 固化安全文本（后续 API 必须复用）：
   - “本系统仅提供劳动纠纷信息辅助，不构成正式法律意见。复杂案件请咨询律师。”

## 常见问题

1. 为什么第一讲不写代码？
   - 因为法律类 Agent 先定边界再写代码，后续可避免大面积返工。
2. 为什么要强调非目标？
   - 工程项目最常见失败原因是边界漂移，非目标能防止需求失控。
