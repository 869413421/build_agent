# 教程黄金标准（跨会话强制基线）

本文件用于锁定“第二章质量标准”，防止换会话后教程质量回退。

## 目标读者与文风

1. 面向工程师读者，不面向模型。
2. 开篇必须有真实痛点与读者收益，不写空泛背景。
3. 语言直接、可执行、少空话。

## 内容结构（必须完整）

1. 本章要解决什么问题（业务痛点 + 工程痛点）
2. 为什么这样设计（设计动机与取舍）
3. 手把手操作步骤（创建目录、创建文件、写代码、运行命令）
4. 图示解释（Mermaid，确保兼容）
5. 测试验证与常见报错处理
6. 本章 DoD 与下一章钩子

## 代码讲解标准

1. 不能只贴代码，必须解释关键字段“为什么存在”。
2. 注释必须解释设计意图、边界条件、失败模式。
3. 代码片段与仓库当前文件保持一致，禁止教程与代码漂移。

## 命令与环境标准

1. Python 命令优先使用 `uv`。
2. 教程必须包含环境准备与缺包兜底。
3. 命令必须可复制运行，不能靠读者自行脑补。

## 图示标准

1. 每章至少 1 张图（推荐 2 张）。
2. 优先使用兼容性更高的 `flowchart`。
3. 图中节点名要和代码对象名对应。

## 质量红线（出现任一项即不合格）

1. 只有步骤没有设计解释。
2. 只有理论没有可运行命令。
3. 使用绝对路径。
4. 代码更新后教程未同步。
5. 无法通过读者视角直接复现。
