# 教程黄金标准（跨会话强制基线）

## 目标

本文件用于锁定“从0到1可独立搭建”的教程质量，防止换会话后质量回退。

## 适用范围

1. `docs/tutorials/01..14` 全部章节
2. `examples/from_zero_to_one/chapter_01..14` 章节快照
3. `src/agent_forge` 主线代码

## 必须满足的结构

每一章必须包含以下 7 段，且顺序一致：

1. 目标
2. 前置条件
3. 章节快照目录
4. 实施步骤（先面后点）
5. 运行命令
6. 验证清单
7. 常见问题

## 代码与讲解标准

1. 代码片段必须完整可运行，不允许“省略实现/见仓库”。
2. 所有关键代码后必须有“代码讲解”：
3. 代码讲解必须覆盖：
设计动机、工程取舍、边界条件、失败模式。
4. 教程中的路径必须可点击，且指向仓库真实文件。
5. 教程代码与仓库代码必须同步；任一改动都要双向更新。

## 章节快照标准

1. 每章必须有独立可运行快照目录：`examples/from_zero_to_one/chapter_XX/`。
2. 快照目录必须包含本章完整代码与测试。
3. 快照需支持“从空目录搭建”场景，命令可复制执行。
4. 每章必须给出“主线同步步骤”（快照到 `src/agent_forge` 的映射）。

## 命令与环境标准

1. Python 命令统一使用 `uv`。
2. 缺包兜底命令按“环境段落策略”按需出现（非每章强制）。
3. 章节命令必须在 Windows PowerShell 下可执行。

## 质量红线（任一命中即不合格）

1. 只有理论，没有可运行命令。
2. 代码不完整或无法独立运行。
3. 章节快照缺失或无法运行。
4. 路径不可点击或指向错误位置。
5. 代码更新后教程未同步。
6. 章节不满足“先讲面，再讲点”。
7. 文章呈现“说明书腔/模板腔”，缺乏真实工程场景与经验表达。
8. 缺失承上启下段落，读者看不出本章与上一章/下一章的链路关系。

## 文风黄金规则（强制）

1. 像真实技术博主写作，不像参数说明书。
2. 每章至少包含一个真实踩坑或失败场景，解释“为什么这样做”。
3. 每个关键步骤必须有工程语境：问题是什么、为何当前解法更稳、代价是什么。
4. 结尾必须明确承上启下：本章解决了什么，下一章为什么必须做什么。

## 学习曲线控制标准（新增，强制）

1. 每章在进入代码前，必须先提供“术语白话卡片”：把核心术语翻译为日常语言，并给出一个完整生活化例子串联主流程。
2. 教学顺序必须固定为：先最小可跑（最短命令）-> 再完整回归 -> 最后深入源码与工程取舍。
3. 复杂章节必须显式给出“阅读路径”，至少包含：先看哪张流程图、先看哪组测试、再看哪几个核心函数。
4. 禁止一上来堆砌抽象概念；若引入新概念，必须在首次出现处用一句话解释“它是干什么的”。

## 环境段落策略（修订）

1. “环境准备与缺包兜底步骤”不再要求每章重复出现。
2. 仅在以下场景必须出现环境段落：
   - 第一章（从零搭建）
   - 本章新增或变更依赖
   - 本章命令与上章存在明显差异，读者可能无法直接复用环境
3. 若本章不含环境段落，必须在“前置条件”中明确复用上一章环境，并给出一条最小验证命令。
## 创建目录与文件命令（硬标准）

1. 每章必须提供 创建目录 + 创建文件 命令块，且在实施步骤前给出。
2. 命令块必须同时提供 Bash 与 Windows PowerShell 两个版本。
3. 命令必须覆盖本章新增的全部真实文件路径，禁止只给 mkdir 不给 touch/New-Item File。
4. 命令必须按教程步骤分段给出，明确“走到哪一步创建哪一个”，禁止一口气全量创建。
5. 命令中的路径必须与仓库实际文件路径一致，可直接复制执行，不允许伪路径。
## 文件前置命令规范（新增硬约束）

1. 每个“文件：`path`”段前必须紧邻创建命令代码块。
2. 创建命令必须使用代码块展示，且固定两段：`bash` 与 `powershell`。
3. 仅需创建该文件本身（`touch` / `New-Item -ItemType File`），禁止在此处插入批量目录脚本。
4. 路径必须与该文件段声明路径完全一致，不允许手写近似路径。
## 章节承接规范（新增硬约束）

1. 除第一章外，每章开篇必须提供“承接上章快照复制”步骤。
2. 必须同时提供 `bash` 与 `powershell` 两段代码块命令。
3. 命令固定语义：将 `examples/from_zero_to_one/chapter_XX` 复制为 `chapter_YY` 后再继续本章改造。
4. 该步骤必须放在本章实施步骤之前，确保读者在上一章可运行基线上继续演进。
