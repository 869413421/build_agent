# 生产就绪清单

## 目标

本文件用于判断框架是否达到“可投入真实生产环境”的最低标准。

当前结论：`未达标`

## 一、当前可确认已具备的基础

以下能力已具备工程原型意义上的基础闭环：

1. `AgentApp -> Agent -> AgentRuntime -> EngineLoop` 主入口链路已成形。
2. `Safety / Tool / Retrieval / Memory / Evaluator` 已接入主链路。
3. 工具失败、memory 写回、幂等缓存、并发记录等关键语义已做过多轮质检修复。
4. 当前全量回归通过：`183 passed`。

这说明项目已经是“可运行的框架原型”，但还不是“可生产交付的系统”。

## 二、当前阻塞生产的关键缺口

### 1. 应用交付层未完成

当前缺口：

1. 没有正式完成的 CLI 业务入口。
2. 没有正式完成的 HTTP API 执行入口。
3. 没有对 `AgentApp` 的统一服务化封装。

生产要求：

1. 至少要有稳定 CLI。
2. 至少要有稳定 API。
3. 两者必须共用同一套 `AgentApp` 装配逻辑。

### 2. 配置与发布门禁未完成

当前缺口：

1. 缺少统一 preflight 检查。
2. 缺少环境变量与运行配置校验。
3. 缺少“发布前必须通过哪些项”的自动门禁。

生产要求：

1. 提供 `preflight` 命令。
2. 配置错误必须 fail fast。
3. 发布前检查结果必须结构化。

### 3. 部署与运维闭环未完成

当前缺口：

1. 尚未完成 Dockerfile / Compose 交付闭环。
2. 缺少环境注入规范。
3. 缺少启动、停止、排障文档。

生产要求：

1. 本地 Compose 可启动。
2. 健康检查可用。
3. 至少有最小运维交接文档。

### 4. 生产稳定性验证未完成

当前缺口：

1. 目前主要是单测与定向集成测试。
2. 没有压力测试。
3. 没有长时间运行测试。
4. 没有故障注入测试。

生产要求：

1. 有基本压测。
2. 有并发与资源稳定性验证。
3. 有外部依赖失败时的降级与告警策略验证。

### 5. 安全与治理闭环未完成

当前缺口：

1. 尚未看到认证鉴权设计。
2. 尚未完成限流、配额、租户隔离策略。
3. 审计、日志、指标虽有基础，但还没有生产治理基线。

生产要求：

1. API 层必须有鉴权预留或正式实现。
2. 多租户隔离必须在运行与存储层都可验证。
3. 安全、审计、告警要能形成完整闭环。

## 三、发布口径建议

当前对外口径建议固定为：

1. `alpha`
2. `内部验证中`
3. `不承诺 production-ready`

不建议使用的对外口径：

1. `production-ready`
2. `可直接商用`
3. `教程已全面对齐并可稳定复现`

## 四、进入生产前的最小停线标准

只有当以下项目全部满足时，才允许重新评估“可生产”：

1. CLI 完成并通过回归。
2. HTTP API 完成并通过回归。
3. Docker Compose 本地交付完成。
4. preflight 配置门禁完成。
5. README 与 01-12 教程全部完成对齐。
6. 至少完成一轮基础压测与异常注入验证。
7. 形成最小运维交接文档。

## 五、当前建议

当前最合理的推进顺序是：

1. 先统一同步教程和接口文档。
2. 再补应用交付层。
3. 然后补发布门禁与部署。
4. 最后再谈生产可用。
